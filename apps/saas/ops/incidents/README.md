# Incident Docs

This folder stores incident-specific documents.

Suggested convention:

- `YYYY-MM-DD-<short-slug>.md`
- Start each file from `postmortem-template.md`

Process:

1. Create incident file when alert is acknowledged.
2. Fill timeline during the incident.
3. Complete root cause and CAPA within 48h.
4. Link runbook and alert/rule changes.
